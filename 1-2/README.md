第四题：
var没有块级作用域，有变量提升现象，可以声明重复的变量名。
let和const有块级作用域，没有变量提升现象，不能在同一作用域中声明重复的变量名。
const使用必须初始赋值，且不能修改。

第五题：

因为fn(){}为普通函数，setTimeout中为箭头函数，所以setTimeout中的this指向的是fn(){}的作用域。

第六题：
Symbol()用于返回一个唯一的值。
可以在想要扩展某对象，但对该对象的key值不熟悉，怕出现命名冲突时使用。
可以作为对象的私有属性key值，想访问key为Symbol()的属性值不容易。

第七题：
浅拷贝仅仅把对象的引用位置赋给新变量，所以浅拷贝新老变量修改会影响彼此。深拷贝则是申请了新的内存地址存放对象数据，深拷贝新老变量修改不会彼此影响。

第八题：
TypeScript是JavaScript的扩展，加入了变量类型系统、Es6语法支持，以及其他主流强类型语言中的类、抽象类、接口、枚举、访问控制符等概念。

第九题：
优点：TypeScript让js开发接近主流强类型语言，类型检查让数据更严谨，类、抽象类、接口让面向对象开发更方便，可以让java之类主流强类型语言程序员转前端更方便。
缺点：增加了额外的学习成本和编码量，对于小项目来说不一定更有效率，更适用于大项目、团队开发项目。

第十题：
引用计数法的优点：回收速度快。
缺点：无法回收循环引用对象，性能消耗大。

第十一题：
标记整理算法先标记与根元素有引用关系的活动对象，再标记没有引用关系的垃圾对象，然后整理活动对象、垃圾对象的内存位置，最后清除垃圾对象。

第十二题：
V8引擎会分配一个小内存空间用于处理新生代对象，然后平分为from和to两个空间，在from空间中进行标记整理处理，然后把活动对象拷贝到to，然后交换from和to空间完成释放。

第十三题：
增量标记算法是在程序运行中穿插着进行的。
增量标记把遍历、标记、清除中的标记阶段碎片化，不一次性执行完毕，而是和程序运行交替进行，这样极大的降低了回收操作对程序运行造成的影响。
