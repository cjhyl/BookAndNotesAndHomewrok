第一题：
前端工程化主要目标是为了使用各种手段提高项目开发效率，统一代码风格质量，降低代码模块的交接、维护难度。以前就遇到过一些问题：

1、需要等后端开发完成后才能对接数据渲染页面，虽然可以自己本地写模拟数据来测试页面，但却没有请求的过程，现在使用mockjs可以模拟请求并返回模拟数据。
2、根据需求做功能模块时，各个功能模块中dom元素、样式表、js逻辑都不一样，标准不好统一，各个模块的结合，标准的整理挺费事儿的。用了vue后，在框架的束缚中写功能模块，各个组件模块排列整齐，逻辑清晰，使用方便。
3、团队合作时，每个人的代码风格不一样，规范不统一，如果某个同事离职，要去维护其代码，要是其没有良好的代码习惯，那真是一个地狱。能用插件规范团队代码风格、结构，对于代码的移交、维护是很有好处的。



第二题：
创建了项目结构后，也提供了项目的代码风格、模块依赖、工具配置的各种规范、约定，提供一个健康的项目基础骨架。
